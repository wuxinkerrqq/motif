# editing_skill.md
> 适用 Agent：ReAct Edit Planner
> 作用：指导精确镜头选择、切点设计、动作衔接、变速处理等执行层面的剪辑技巧

---

## 核心原则

**选片的任务是执行，不是重新规划。**

叙事大纲已经决定了"用哪个段落、配什么情绪、何时爆发"，选片时要做的是：
1. 在 `scene_table` 中找到最匹配的具体场景
2. 确保前后镜头在视觉和情绪上连贯自然
3. 每一个决策都必须有依据，不能随机选片

---

## 一、镜头选择的优先级规则

在候选场景中做最终选择时，按以下优先级依次判断：

```
优先级 1：情绪匹配（editing_metrics.emotion_mood 和音乐段落 mood 一致）
优先级 2：能量匹配（editing_metrics.action_density ≈ 音乐段落 energy）
优先级 3：动作连贯性（和上一个镜头的运动方向/动作类型衔接自然）
优先级 4：视觉焦点位置（和上一个镜头的视觉中心接近）
优先级 5：时长适配（scene.duration 能填满音乐段落，或可通过 speed_factor 调整）
```

当优先级之间冲突时，**情感永远第一**（沃尔特·莫奇法则第一条）。

## 二、切点设计

### 2.1 切在动作中（Cutting on Action）

在人物或物体运动过程中切换镜头，而不是在动作完成后。

- 人物抬手、转身、奔跑、攻击等动作进行中，在动作顶点切换
- 不要等动作完全停止再切，那会让剪辑感"很重"
- 适用于所有 `action_density >= 4` 的素材

### 2.2 Beat Sync（节拍同步切换）

```
energy >= 7（drop/chorus）  → 强拍切换，精确对齐 beat_array
energy 4-6（verse/buildup） → 弱拍或强拍切换均可
energy <= 3（intro/outro）   → 不强制卡拍，让镜头自然呼吸
```

beat snap 由代码自动执行（容忍范围 ±0.15s），你只需要选好场景和大致时间点。

### 2.3 Match Cut（匹配剪辑）

两个镜头在视觉构图、形状或动作上高度相似，通过匹配制造视觉连贯性：

- **动作匹配**：不同人物做出相同或相似动作（向右挥拳 → 向右挥剑）
- **图形匹配**：视觉构图、形状、颜色高度相似（圆形月亮 → 圆形眼睛特写）
- **主题匹配**：情绪或叙事主题呼应（孤独站立的人物 → 空旷的景别）

## 三、动作衔接规则

### 3.1 运动方向一致性

相邻两个镜头的主要运动方向应保持一致，避免方向突然反转造成视觉混乱。

参考字段：`scene.visual_elements.camera_movement`

```
正确：向右奔跑的人物 → 向右冲刺的另一个人物
错误：向右奔跑的人物 → 向左奔跑的另一个人物（方向反转，混乱）
```

**例外：** 转折点可以刻意反转方向表达冲突；`key_change` 或 `explosion_after_silence` 时，方向反转配合情绪反转效果震撼。

### 3.2 视觉焦点引导

相邻两个镜头的视觉焦点应在屏幕上的位置接近，避免视线跳跃过大：

- 中心焦点 → 任意焦点（中心是万能过渡）
- 左侧焦点 → 左侧或中心（不要直接跳到右侧）
- 右侧焦点 → 右侧或中心（不要直接跳到左侧）

### 3.3 相似性衔接

即使两个镜头来自完全不同的作品，通过选择相似的场景制造连贯感：

| 维度 | 参考字段 |
|------|---------|
| 情绪相似 | `editing_metrics.emotion_mood` |
| 能量相似 | `editing_metrics.action_density`（差值 ≤ 2） |
| 色调相似 | `visual_elements.dominant_color` |
| 运动方向相似 | `visual_elements.camera_movement` |

## 四、变速处理（speed_factor）

### 4.1 使用场景

```
片段时长 < 目标时长 × 0.7  → 放慢（speed_factor < 1.0），拉伸画面
片段时长 > 目标时长 × 1.3  → 加速（speed_factor > 1.0），压缩画面
时长接近                   → 轻微变速 ±20%，几乎察觉不到
```

### 4.2 变速的艺术用法

**慢入快出（speed_factor 从 0.6 渐升到 1.2）**
- 制造"蓄力后爆发"的视觉感，对应 buildup 段落

**快入慢出（在关键时刻突然降速）**
- 强调最重要的单帧画面，对应 drop 触发瞬间

### 4.3 变速限制

- `speed_factor` 范围：0.3x（极慢）～ 3.0x（极速），超出范围画质明显下降
- 变速后实际使用的素材时长 = `scene.duration / speed_factor`，不能超过 `scene.duration`

## 五、镜头时长控制

| 音乐段落类型 | energy 范围 | 建议单镜头时长 |
|------------|------------|--------------|
| intro/outro | 0–3 | 3–8 秒 |
| verse/bridge | 3–6 | 2–4 秒 |
| buildup | 5–7 | 1–3 秒，逐渐缩短 |
| chorus | 6–8 | 1–2 秒 |
| drop | 8–10 | 0.5–1.5 秒 |

长短交替比均匀切换更有冲击力：长-短-短-长-短 的节奏优于 短-短-短-短。

## 六、常见错误

| 错误类型 | 表现 | 修正方法 |
|---------|------|---------|
| 情绪断层 | 悲伤镜头后接欢快镜头无过渡 | 优先匹配 `emotion_mood` |
| 节奏断裂 | 高能量段落出现长静止镜头 | 放宽 mood 约束但保持 `action_density` |
| 重复画面 | 同一场景出现超过一次 | 工作记忆中已记录已用场景，主动避免 |
| 切点脱拍 | 镜头切换和音乐节拍明显错位 | beat snap 容忍范围 ±0.15s，选好大致时间点即可 |
| 变速过度 | 素材被拉伸到 0.2x，画面卡顿 | 换一个时长更接近的素材 |
| 方向跳切 | 相邻镜头运动方向突然反转 | 检查 `visual_elements.camera_movement` |
